üì¶ Catalogs and Products Database

A Product Catalog is a collection of classified products organized by type. These catalogs are stored in a shared database and can be browsed through various pages within the app. However, only products from the User Palette can be used during modeling or scripting.

üîç Accessing the User Palette

You can open the User Palette from the following places:

  • On the main page, via the left-side panel under the Solution Tree
  • From the Database Page
  • From the Catalog Page ‚Äî by selecting an element, right-clicking, and choosing "Catalog"

‚≠ê Active Product

Each product type can have an active product. When you use the program menus to add an item, the active product for that type will be used. You can also drag and drop any product from the User Palette into your model.

üìÅ Types of Catalogs

  • Default Catalogs: Always present and cannot be removed (e.g., Generic Doors, Windows).
  • Optional Catalogs: Large categories (e.g., beams, columns) not loaded by default. Users can add or remove them.
  • User Catalogs: Created by the user. These can be removed and are deleted when the database is reset.

Important: After making changes, always click ‚ÄúUpdate User Palette‚Äù to apply them.

‚ûï Creating Products for Custom Catalogs

You can add products to custom catalogs using two methods:

  1. Create a product in the app, right-click the element or press I, then choose ‚ÄúSave as Product‚Äù from the catalog page menu.
  2. Use Lua API commands (see documentation and GitHub examples).

Note: Resetting the database via the UI restores default data. Resetting via Lua command ResetDataBase clears all entries, leaving the database empty.

Example: reset and repopulate using Lua:


ResetDataBase()
AddCatalogToDB("MyCatalog")
ResetUserPalette()
